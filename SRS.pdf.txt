2. System Overview
The AI-Powered Analytics Intelligence Platform is a multi-tenant SaaS web application. It connects to all four 
data sources — Google Analytics 4 (GA4), Google Search Console (GSC), Google Ads API, and Facebook Ads 
(Meta Marketing API) — fetches aggregated performance data, and uses Anthropic Claude AI with dynamic 
prompt engineering to convert raw cross-channel metrics into clear, human-readable insights in one unified 
dashboard.

6. Authentication Implementation
6.1 Email/Password Auth
• bcrypt with saltRounds = 12 for all password hashing
• JWT signed with HS256 — secret from env var JWT_SECRET — expiry 7 days
• JWT payload: { userId, email, iat, exp } — nothing else
• Email verification: crypto.randomUUID() → saved to user.emailVerifyToken → link sent → on click set 
emailVerified: true, clear token
• Password reset: UUID → passwordResetToken + passwordResetExp (now + 1hr) → link sent → validate 
token + expiry → hash new password → clear fields
• Account lockout: loginAttempts >= 5 → lockUntil = Date.now() + 15 min → reject with 423 Locked
• On successful login: reset loginAttempts = 0, clear lockUntil
6.2 Google OAuth  (GA4 + GSC + Google Ads)
• Strategy: passport-google-oauth20
• Scopes: https://www.googleapis.com/auth/analytics.readonly  
https://www.googleapis.com/auth/webmasters.readonly  https://www.googleapis.com/auth/adwords  
openid  email  profile
• accessType: 'offline'  —  required to receive refresh_token
• prompt: 'consent'  —  required to get refresh_token on every auth, not just first time

• All three Google services (GA4, GSC, Google Ads) share the same GoogleToken — one OAuth connection 
covers all three
6.3 Facebook OAuth  (Meta Ads)
• Strategy: passport-facebook with profileFields: ['id', 'displayName', 'email']
• Scopes: ads_read, ads_management, business_management, email, public_profile
• Exchange short-lived token for long-lived token immediately after OAuth callback using Graph API
• Long-lived token endpoint: GET /oauth/access_token?
grant_type=fb_exchange_token&client_id=&client_secret=&fb_exchange_token=
• Save encrypted long-lived token + expiresAt to FacebookToken document
• On callback: find or create User by facebookId → upsert FacebookToken → sign JWT → redirect to 
/auth/callback?token=JWT
6.4 Token Encryption  —  /server/utils/encrypt.js
• Algorithm: AES-256-GCM
• Key: 32-byte Buffer from env var ENCRYPTION_KEY (64 hex chars)
• Format stored in DB: iv:authTag:encryptedData  —  all as hex strings joined by colon
• Export: encrypt(plaintext) → string  and  decrypt(ciphertext) → string
• Apply to: GoogleToken.accessToken, GoogleToken.refreshToken, FacebookToken.accessToken
