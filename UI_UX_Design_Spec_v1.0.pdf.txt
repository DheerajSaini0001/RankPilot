1. Design Principles
These principles govern every design decision across all screens. Every component, layout, and interaction must 
be evaluated against them before implementation.

Professional Trust The platform handles business

2. Color Palette
All colors are defined as Tailwind CSS custom colors in tailwind.config.js. Use ONLY these values throughout the 
entire application. Never use arbitrary hex values in component files.

elements.

  brand-600 — Primary Blue  #2E75B6  |  Primary buttons, active nav, links, chart primary line

3. Typography
The platform uses Inter as the primary font. It is clean, highly legible at small sizes, and widely used in enterprise 
SaaS products.
3.1 Font Setup
• Font family: Inter (Google Fonts)
• Import in index.html: <link href='https://fonts.googleapis.com/css2?
family=Inter:wght@300;400;500;600;700&display=swap' rel='stylesheet'>
• Set in tailwind.config.js: fontFamily: { sans: ['Inter', 'ui-sans-serif', 'system-ui'] }
• Monospace (for code/API keys): font-mono — Tailwind default

4.7 Skeleton Loader
• Every card, chart, and table must show a skeleton loader while data is fetching
• Skeleton: animate-pulse bg-neutral-200 rounded dark:bg-neutral-700
• KPI card skeleton: 3 gray bars at heights h-4, h-8, h-4 with spacing
• Table skeleton: 5 rows of h-10 bars with full width
• Chart skeleton: single h-48 bar filling the chart container
• Never show empty white space — skeleton must appear within 50ms of component mount

8.2 Dark Mode Implementation
• Set strategy in tailwind.config.js: darkMode: 'class'
• Toggle: document.documentElement.classList.toggle('dark')
• Persist in localStorage key: 'theme' — values: 'light' | 'dark' | 'system'
• System: window.matchMedia('(prefers-color-scheme: dark)').matches
• Apply on app load before render to prevent flash of wrong theme
• All chart colors must also change in dark mode — use Recharts custom colors from Zustand theme
